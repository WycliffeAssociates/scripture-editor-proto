# Manual verification of cursor correction feature

Perform manual verification of the cursor correction feature in the actual editor application. Test real-world scenarios and edge cases to ensure the feature works as intended in the browser environment.

Test scenarios:
1. Start editor in Regular (WYSIWYG) mode
2. Navigate cursor onto a locked marker node (e.g., \v, \p, \q1)
3. Verify cursor automatically moves to nearest editable position (forward first, then backward)
4. Test in USFM mode - cursor should NOT move (markers are editable)
5. Test in Source mode - cursor should NOT move
6. Test with empty document - no errors, no-op
7. Test at document start - cursor moves forward
8. Test at document end - cursor moves backward
9. Test with only locked nodes - cursor stays in place
10. Performance test with large chapters (100+ verses)

Launch:
- pnpm dev.web for browser testing
- Or pnpm dev.tauri for desktop testing

Acceptance criteria:
- Cursor never trapped in locked marker in Regular mode
- Cursor movement is automatic and seamless
- Feature disabled in USFM and Source modes
- No jank or lag in large documents
- No console errors or warnings
- Visual inspection confirms cursor positioning is natural

If issues found:
- Document in ticket notes
- Create follow-up bug ticket if needed

Acceptance:
- Cursor correction works in Regular (WYSIWYG) mode
- No cursor trapping in locked markers
- Feature properly disabled in USFM and Source modes
- Performance acceptable on large chapters
- No console errors
- Edge cases verified (boundaries, empty docs)
