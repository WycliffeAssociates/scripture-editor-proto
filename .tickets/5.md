# Refactor USFMPlugin: Decompose into focused hooks

The USFMPlugin.tsx component has become a 'God Object' responsible for too many distinct concerns: linting, document structure maintenance, input handling, view updates, and cleanup coordination. This makes it hard to read, test, and maintain.

This epic splits USFMPlugin into smaller, focused custom hooks that manage their own Lexical listeners.

Proposed Structure:
1. useEditorLinter - Handling linting logic
2. useEditorStructure - Maintaining document structure and metadata
3. useEditorInput - Handling keyboard commands and transforms
4. useEditorView - Handling visual-only updates (WYSIWYG preview toggling)

Reference: plans/current/2026-01-12-refactor-usfm-plugin.md

## Design

Create new directory: src/app/domain/editor/hooks/
Each hook will accept editor: LexicalEditor and use useWorkspaceContext internally for dependencies.
The main USFMPlugin will become a thin composition layer that orchestrates the four hooks.

## Acceptance Criteria

USFMPlugin.tsx is refactored into four focused hooks. All functionality preserved. Unit tests pass. E2E tests pass. Manual verification confirms linting, marker visibility, reference pane sync, and cursor correction all work correctly.
