# Run pnpm knip to check for structural issues

Execute Knip to ensure the refactoring hasn't introduced any structural issues like unused exports, unlinked files, or circular dependencies.

Steps:
1. Run pnpm knip
2. Review any warnings or errors
3. Fix any issues found:
   - Unused exports in the new hook files
   - Unlinked files (if any were accidentally created)
   - Circular dependencies between hooks
   - Missing dependencies in package.json (unlikely for this refactor)
4. Re-run pnpm knip until all issues are resolved

This ensures the codebase maintains good structural hygiene after the refactoring.

Dependencies: Must wait for USFMPlugin.tsx update (ticket 5.6).

## Design

Knip should ideally report no issues after the refactoring.
If there are unused exports, they should be removed or documented as intentionally exported.

## Acceptance Criteria

pnpm knip runs without errors or warnings. No unused exports, unlinked files, or circular dependencies.
