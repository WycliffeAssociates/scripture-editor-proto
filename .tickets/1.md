# Refactor Plan: Hardened Test IDs

Centralize all dynamic Test ID generation in src/app/data/constants.ts and enforce usage across the app. This plan aims to replace ad-hoc string concatenation with a centralized TEST_ID_GENERATORS object to improve maintainability and reduce brittleness in tests.

## Design

Follow the plan to add TEST_ID_GENERATORS to constants.ts, refactor components to use these generators, and update E2E tests accordingly.

## Acceptance Criteria

All dynamic test IDs are centralized in TEST_ID_GENERATORS, components are refactored to use these generators, and E2E tests are updated and passing.
