# Manual verification of refactored editor

Perform manual testing to verify all editor features work correctly after refactoring. This is the final verification step before closing the epic.

Verification checklist:
1. **Linting:**
   - Create an intentional error (e.g., missing closing marker)
   - Verify lint warning appears in the editor
   - Verify lint warning disappears when fixed

2. **Marker visibility:**
   - Set markersViewState to WHEN_EDITING
   - Start typing in WYSIWYG mode
   - Verify markers hide while typing
   - Stop typing and verify markers reappear

3. **Reference pane:**
   - Open reference pane (if available)
   - Move cursor to different verses
   - Verify reference pane scrolls to match current verse

4. **Cursor correction:**
   - Type near immutable markers (e.g., \v)
   - Verify cursor doesn't jump inside the marker
   - Verify cursor positions correctly after marker

5. **Immutable markers:**
   - Try to type inside an immutable marker (e.g., \v 1|)
   - Verify cursor is blocked or redirected
   - Try to paste content with markers
   - Verify paste behavior is correct

6. **Document structure:**
   - Add a new chapter marker
   - Verify chapters/verses are numbered correctly
   - Verify metadata is maintained

7. **Mode switching:**
   - Switch from WYSIWYG to SOURCE mode
   - Verify no effects run in SOURCE mode
   - Switch back to WYSIWYG
   - Verify all effects resume

8. **Performance:**
   - Type rapidly
   - Verify no lag or stutter
   - Verify throttling/debouncing is working (check console for timing logs)

Dependencies: Must wait for E2E tests (ticket 5.8).

## Design

Manual testing should be done in the Tauri desktop environment for full coverage.
Use a real USFM file (e.g., from src/test/mockData/).

## Acceptance Criteria

All 8 verification areas pass. Editor behavior matches pre-refactor behavior. No user-facing regressions found. Performance is acceptable (no noticeable lag).
