# Run E2E editor tests to verify refactoring

Execute end-to-end test suite specifically for editor functionality.

Steps:
1. Run pnpm test:e2e with filter for editor tests
2. Verify editor creation and initialization
3. Test typing behavior:
   - Type normal text
   - Type near markers
   - Try to edit immutable markers
4. Test linting:
   - Create intentional error
   - Verify lint warning appears
5. Test marker visibility:
   - Toggle markers in preview mode
   - Verify they hide/show correctly
6. Test reference pane synchronization:
   - Open reference pane
   - Verify it scrolls with editor cursor
7. Test cursor correction:
   - Type near markers
   - Verify cursor doesn't jump unexpectedly
8. Test document structure:
   - Add verses/chapters
   - Verify document structure is maintained
9. Test mode switching:
   - Switch between SOURCE and WYSIWYG modes
10. Fix any E2E test failures caused by the refactoring

If tests fail, check:
- Hook cleanup functions are returning correct unregister functions
- Event listeners are being registered at mount time
- Mode checks (SOURCE vs WYSIWYG) are working
- Tag filtering is working correctly

Dependencies: Must wait for unit tests (ticket 5.7).

## Design

E2E tests should not require changes - they test user-facing behavior which should be preserved.
If E2E tests fail, it indicates a behavioral regression in the refactored code.

## Acceptance Criteria

All E2E editor tests pass. No regressions in user-facing behavior. All editor features work correctly in browser tests.
